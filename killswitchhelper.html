Debian based distributions
Terminal
  1. Open a terminal and paste the following command to add my Personal Package Archive: 
                                
                              sudo add-apt-repository ppa:eun-s/ppa
  2. After the PPA has been added, proceed to install with the following commands: 
                                
                              sudo apt-get update
                              sudo apt-get install killswitchhelper
Debian based distributions
Synaptic Package Manager

Open the Synaptic Package Manager and go to the Repositories area.

Like in the picture above click New, choose Binary (deb) and paste in the URI field:

                          
                            http://ppa.launchpad.net/eun-s/ppa/ubuntu/

In the Distribution field, paste:

                          
                            xenial

And in the Section(s) field:

                          
                            main

Then click OK. If you want the source code too you follow the same steps again with the same information, but instead of choosing Binary (deb) from the dropdown, you choose Source (deb-src). Finally, Reload the repositories and query for killswitchhelper. Then you can install it from there.

Source
Terminal
  1. Make sure you have the following programs and dependencies to compile adequately:
    • cmake
    • make
    • valac (version >= 0.34)
    • elementary-sdk
    • valac-0.34-vapi
    • libgtk-3-dev
    • libpolkit-gobject-1-dev
    2. If you are in an elementaryOS release it is very likely for all of those packages to be found in Synaptic. Otherwise you may need to add the following PPA to your repositories:

    http://ppa.launchpad.net/elementary-os/os-patches/ubuntu/

  2. Make sure you have the following dependencies to then be able to use the application:
    • libgtk-3-0 (version >= 3.18)
    • libpolkit-gobject-1-0
  3. Download the source code for this application here.
  4. Extract it anywhere in your computer with your preferred extraction tool.
  5. Open a Terminal and enter the directory named build.
  6. Once the terminal is in that directory paste the following command to create the make files: 
                                cmake -DCMAKE_INSTALL_PREFIX=/usr ../
  7. Type the following command in the terminal and press enter: 
                                make
  8. Finally, execute the following command: 
                                sudo make install

Note that this source code comes with an uninstall script. To uninstall, while being in the build directory, execute the following command: 

                          sudo make uninstall

ðŸ”— About

An application to easily prepare a Kill Switch with a graphical interface.

ðŸ”— How to use

  1. Launch the application
  2. Look at the generated routes
  3. Connect to your VPN service
  4. Hit the refresh button in this application
  5. Look at the new routes and remove the ones you don't want the operating system using in case the VPN connection fails.

ðŸ”— Good to know

  • When the routing table is populated, the first one in the list is the one the operating system uses for networking.
  • If this first one fails to connect to the internet for whatever reason, the operating system will automatically use the next route in the list if there is one.
  • If the next used route for networking fails too, the operating system continues looking for the next and so on.
  • This process tends to happen transparently to the user. Meaning that if you are using a VPN for networking and that connection fails for whatever reason, the operating system will use the next route which will reveal your real connection.
  • If this application is used properly and the adequate routes are removed while connected to the VPN service, you can avoid that from happening since as soon as the VPN connection fails, there will be no connection to the internet.

ðŸ”— Troubleshooting

  • Sometimes, you may get an error message like this

    It means either one of these

    1. The authentication failed - make sure you enter the correct password that gives you root access.
    2. The authentication was dismissed by you, the user - try again and don't dismiss the authentication dialog.
    3. The program helper could not be found in the system path - make sure the binary killswitchhelperhelper is in the system path. The normal place of installation is in /usr/bin. Sometimes, when compiling from source code and installing with make install, some files fail to install in their respective places. Try running make install again if it was that you installed using this method of compiling the source yourself.
    4. The route you are trying to add was not added because it could not be added according to routing rules - Some routes require a netowrk route prefix to exist before adding them. That means that if you removed a route similar to this: 192.168.1.0/24 (note the slash), you won't be able to create a route to your default gateway. Try re-adding the network prefix route you eliminated before adding your default gateway back.
  • Other times you may launch the application and nothing will happen. No user interface is shown and the process exits. In such cases, it can be that the application cannot find the user interface file to load it. The user interface will always be looked for in the same directory the executable of the application lives or in /usr/share/KillSwitchHelper/KillSwitchHelper.glade. Make sure the file KillSwitchHelper.glade is located in such directory.
